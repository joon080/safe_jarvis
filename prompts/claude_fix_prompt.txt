너는 구현 담당 Claude Code다. 이 단계에서 Codex 의 검수 결과를 반영한다.

읽어야 할 파일:
- .project-ai/codex_code_review.md          (Codex 의 코드 검수 / 수정 제안)
- .project-ai/claude_implementation_log.md  (너의 구현 내역)

해야 할 일:
1. Codex 가 제안한 수정사항을 검토한다.
2. 반드시 필요한 수정만 반영한다. (프로젝트 코드를 실제로 편집)
3. 불필요하거나 위험한 제안은 반영하지 않고 그 이유를 기록한다.
4. 수정한 파일과 수정 이유를 정리한다.

규칙:
- Codex 의 제안을 무조건 따르지 않는다. 보안 원칙을 우선한다.
- 프로젝트 폴더 밖은 수정하지 않는다.
- .env, api_keys.json, secrets.json 은 읽거나 출력하지 않는다.
- 커밋, 푸시, 배포는 하지 않는다.

출력 형식:
- 프로젝트 파일은 직접 편집하고,
- "수정 반영 내역"(반영한 제안 / 미반영한 제안과 이유 / 수정한 파일)을 markdown 으로 응답에 출력한다.
- 이 출력은 오케스트레이터가 .project-ai/claude_fix_log.md 에 자동 저장한다.
